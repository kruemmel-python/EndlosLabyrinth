keyboard.js

This file implements the KeyboardJS library, a JavaScript utility for handling keyboard input, allowing developers to bind functions to single keys or complex key combinations, and to manage keyboard-driven axis controls. It supports both AMD and browser global environments.

Module Initialization

The library is wrapped in an Immediately Invoked Function Expression (IIFE) that checks for AMD (Asynchronous Module Definition) support. If AMD is available, it registers itself as an anonymous module. Otherwise, it exposes itself globally as k and KeyboardJS on the context (typically window).

Polyfills

  • bind(target, type, handler)

    A cross-browser event binding utility. It uses addEventListener for modern browsers and attachEvent for older Internet Explorer versions.

    • target: The DOM element to attach the event listener to.
    • type: The event type (e.g., "keydown", "keyup").
    • handler: The function to execute when the event fires.

    Returns: undefined

  • Array.prototype.indexOf

    A polyfill for Array.prototype.indexOf, ensuring compatibility with browsers that do not natively support this method.

Global Variables

  • locals

    An object containing keyboard locale definitions. Currently, it includes a 'us' locale mapping common key names (e.g., "backspace", "tab", "a", "f1") to their respective JavaScript keyCode values.

    //If you create a new local please submit it as a pull request or post it in the issue tracker at // http://github.com/RobertWhurst/KeyboardJS/issues/

  • keys

    The currently active locale's key-to-keyCode mapping. Initialized to locals['us'].

  • activeKeys

    An array that stores the string names of all keys currently being held down.

  • activeBindings

    An object that stores currently active key bindings, keyed by their keyCombo string.

  • keyBindingGroups

    An array of arrays, where each inner array holds key bindings grouped by the number of keys in their combination (e.g., keyBindingGroups[1] for single-key binds, keyBindingGroups[2] for two-key combos).

Event Listeners

The library attaches global event listeners to the document and window objects:

  • document "keydown"

    When a key is pressed, this handler identifies the key using the event.keyCode and adds its string representation to the activeKeys array if it's not already present. It then calls executeActiveKeyBindings(event) to process any matching key bindings.

  • document "keyup"

    When a key is released, this handler removes the corresponding key from the activeKeys array. It then calls pruneActiveKeyBindings(event) to handle bindings that are no longer active.

  • window "blur"

    When the browser window loses focus, this handler clears the activeKeys array, effectively resetting the state of all pressed keys. It also calls pruneActiveKeyBindings(event) to ensure any active bindings are properly ended.

Internal Functions

  • queryActiveBindings()

    Generates an array of active key bindings. It iterates through keyBindingGroups from longest key combinations to shortest, checking if all keys required by a binding are present in the activeKeys array.

    • Parameters: None
    • Returns: Array - An array of active binding objects.

  • executeActiveKeyBindings(event)

    Collects active keys, sets active binds, and fires on key down callbacks.

    • event: The keyboard event object.

    This function queries for active bindings, then iterates through them. For each binding, it checks if its keys have already been "spent" by a higher-priority binding (longer key combo). If not, it executes the binding's callback function, adds the binding to activeBindings, and marks its keys as spent. If any binding is fired, it returns false to prevent default browser actions (like event bubbling).

    • Returns: Boolean - false if a binding was executed and default event action should be prevented, otherwise true or undefined.

  • pruneActiveKeyBindings(event)

    Removes no longer active keys and fires the on key up callbacks for associated active bindings.

    • event: The keyboard event object.

    This function compares the currently activeBindings with the result of queryActiveBindings(). If a binding in activeBindings is no longer found in the currently active set, its endCallback is executed, and the binding is removed from activeBindings.

    • Returns: Boolean - false if an endCallback returned false, otherwise undefined.

Public API Functions

  • bindKey(keyCombo, callback, endCallback)

    Binds an on key down and an optional on key up callback to a key or key combo. Accepts a string containing the name of each key you want to bind to comma separated. If you want to bind a combo the use the plus sign to link keys together. Example: 'ctrl + x, ctrl + c' Will fire if Control and x or y are pressed at the same time.

    • keyCombo: String - A comma-separated string of key combinations (e.g., "a", "ctrl + s", "shift + alt + d").
    • callback: Function - The function to execute when the key combo is pressed and becomes active. Parameters: (event, keys, keyCombo).
    • endCallback: Function (Optional) - The function to execute when the key combo is released or becomes inactive. Parameters: (event, keys, keyCombo).
    • Returns: Object - An object with a clear() method to unbind the created key binding.

  • bindAxis(up, down, left, right, callback)

    Binds keys or key combos to an axis. The keys should be in the following order; up, down, left, right. If any of the the binded key or key combos are active the callback will fire. The callback will be passed an array containing two numbers. The first represents x and the second represents y. Both have a possible range of -1, 0, or 1 depending on the axis direction.

    • up: String - Key combo for the 'up' direction.
    • down: String - Key combo for the 'down' direction.
    • left: String - Key combo for the 'left' direction.
    • right: String - Key combo for the 'right' direction.
    • callback: Function - The function to execute when an axis key is active. Parameters: (axisArray), where axisArray is [y, x].

    This function sets up individual key bindings for each direction and uses a setInterval timer to continuously call the provided callback with the current axis state ([y, x], where -1 is up/left, 1 is down/right, 0 is neutral).

    • Returns: Object - An object with a clear() method to unbind all associated axis bindings and stop the interval.

  • unbindKey(keys)

    Clears all key and key combo binds containing a given key or keys.

    • keys: String - A comma-separated string of keys to unbind. Can also be the string 'all' to clear all bindings.

    This function iterates through all stored key bindings and removes any binding that includes one of the specified keys.

    • Returns: undefined

  • getActiveKeys()

    Gets an array of active keys.

    • Parameters: None
    • Returns: Array - An array of strings representing the currently pressed keys.

  • addLocale(local, keys)

    Adds a new keyboard local not supported by keyboard JS.

    • local: String - The name of the new locale (e.g., "fr").
    • keys: Object - An object mapping key names to their keyCode values.
    • Returns: undefined

  • setLocale(local)

    Changes the keyboard local.

    • local: String - The name of the locale to activate.

    If the specified locale exists in the locals object, it updates the global keys variable to use that locale's key mappings.

    • Returns: undefined

Returned Module Object

The library returns an object containing the public API: 

{
    "bind": {
        "key": bindKey,
        "axis": bindAxis
    },
    "activeKeys": getActiveKeys,
    "unbind": {
        "key": unbindKey
    },
    "locale": {
        "add": addLocale,
        "set": setLocale
    }
}

maze.js

The maze generation module has been significantly expanded. In addition to the legacy generateSquareMaze helper, the file now exposes a MazeGenerator namespace with interchangeable algorithms, path-finding helpers, start/end point selection and simple rendering utilities. All algorithms still return boolean grids (true for a wall, false for a passage) so the existing game code continues to work unchanged.

generateSquareMaze(dimension)

Backwards compatible facade for the classic recursive backtracking generator. It simply proxies to MazeGenerator.generateDepthFirst and keeps the historical dimension property on the returned grid.

MazeGenerator (global namespace)

Provides a high-level API around maze creation and analysis:

  • build(options): Generate a maze with the selected algorithm and optional helpers (automatic start/end placement, solving, ASCII export).
  • generateDepthFirst(options): Recursive backtracking (legacy default).
  • generatePrim(options): Randomized Prim's algorithm for denser, cave-like mazes.
  • generateKruskal(options): Kruskal/Minimum-Spanning-Tree-based generator with uniform passages.
  • generateEller(options): Eller's algorithm for efficient streaming generation row by row.
  • generateHex(options): Prim-style growth on a hexagonal grid, returning axial coordinates and adjacency.
  • chooseEndpoints(grid, strategy): Determine start/end cells (longest-path by default, "corners" as alternative).
  • solve(grid, start, end): Breadth-first solver returning the path between two cells.
  • longestPath(grid): Convenience wrapper that computes the maze's diameter path.
  • toString(grid): Create a text/ASCII representation useful for debugging or logging.
  • drawOnCanvas(canvas, grid, options): Render a maze on an HTML <canvas> with optional colouring for start/end/path.

Generation Algorithms

Every algorithm accepts either a dimension (for square layouts) or explicit width and height. Even values are automatically rounded up to the next odd number to preserve border walls.

  • Depth-first search: Classic recursive backtracking using randomized directions.
  • Randomized Prim: Maintains a frontier of candidate cells, carving towards already-open passages to create organic structures.
  • Kruskal: Builds a minimum spanning tree over the cell graph via a disjoint-set union, producing uniform corridor lengths.
  • Eller: Processes the maze row by row with dynamic set merging, ideal for generating tall mazes or streaming output.

Additional Topologies

generateHex grows a maze on a hexagonal grid described in axial coordinates. Each cell lists its connected neighbours, enabling downstream rendering or simulation on non-square tilings.

Analysis & Utilities

The helper functions unlock new gameplay features and tooling:

  • Automatic endpoints: chooseEndpoints selects two distant cells (or opposing corners) for start/finish placement.
  • Solving support: solve returns the shortest path between two coordinates, enabling hint systems or automated testing.
  • Longest path discovery: longestPath exposes the maze diameter for analytics or challenge modes.
  • Rendering helpers: drawOnCanvas quickly visualises a maze without requiring the full 3D scene graph.

index.html

This file serves as the main entry point for the Astray game, orchestrating the loading of external libraries, defining global game state, managing UI elements, and implementing the core game logic including rendering, physics, and user interaction.

External Script Imports

The following JavaScript libraries are loaded to provide essential functionalities:

  • Box2dWeb.min.js: The Box2D physics engine for collision detection and rigid body simulation.
  • Three.js: The Three.js 3D library for rendering the game world.
  • keyboard.js: A custom script likely handling keyboard input for game controls.
  • jquery.js: The jQuery library for DOM manipulation and event handling.
  • maze.js: A custom script responsible for maze generation and related logic.

A comprehensive set of global variables manages the game's state, rendering, physics, and UI:

  • camera, scene, renderer, light: Core Three.js objects for 3D rendering.
  • mouseX, mouseY: Mouse coordinates (though not actively used for direct control in the provided snippet).
  • maze, mazeMesh: Data structure for the maze layout and its Three.js mesh representation.
  • mazeDimension: The current size of the square maze (e.g., 11x11).
  • planeMesh, ballMesh: Three.js meshes for the ground plane and the player's ball.
  • exitMesh: A Three.js mesh that renders the red exit plate inside the maze.
  • ballRadius: Radius of the player's ball.

  • ironTexture, planeTexture, brickTexture: Three.js textures for the ball, ground, and maze walls.
  • gameState: String indicating the current state of the game (e.g., 'menu', 'initialize', 'play', 'fade in', 'fade out').
  • levelIndex: The current level number the player is on.
  • isPreparingLevel: Boolean flag to prevent multiple simultaneous level preparations.
  • lastMazeName: Name of the last loaded maze.

  • grip:
    • id: 'grip'
    • durationMs: 12000 (12 seconds)
    • apply(): Sets currentFrictionFactor to 0.985.
    • reset(): Resets currentFrictionFactor to baseFrictionFactor.
  • coinrush:
    • id: 'coinrush'
    • durationMs: 15000 (15 seconds)
    • apply(): Sets currentCoinMultiplier to 2.
    • reset(): Resets currentCoinMultiplier to 1.
  • vision:
    • id: 'vision'
    • durationMs: 10000 (10 seconds)
    • apply(): Sets visionBoostActive to true and increases light.intensity.
    • reset(): Sets visionBoostActive to false and resets light.intensity to defaultLightIntensity.

    Box2dWeb.min.js

    This file contains the minified JavaScript implementation of the Box2D physics engine. It defines the core structures, collision detection, common utilities, dynamics (bodies, fixtures, contacts, world), controllers, and joints necessary for 2D physics simulations.

    Global Namespace

    • Box2D

      The primary global object that serves as the namespace for all Box2D components, classes, and utilities.

    Core Utilities and Extensions

      A utility function to establish classical inheritance between two JavaScript constructor functions.

      • Parameters:
        • y: The child constructor function.
        • w: The parent constructor function.

    • Box2D.generateCallback(y, w)

      Generates a callback function that binds a method w to a specific context y.

      • Parameters:
        • y: The context (this value) for the callback.
        • w: The function to be called.
      • Returns:

        function: A new function that, when called, invokes w.apply(y, arguments).

    • Box2D.NVector(y)

      Creates a new array (Vector) of a specified size, initialized with zeros. Used for numerical arrays.

      • Parameters:
        • y: The desired size of the vector.
      • Returns:

    • Box2D.is(y, w)

      Checks if an object y is an instance of a given type w, or if its constructor implements w.

      • Parameters:
        • y: The object to check.

    Collision Module (Box2D.Collision)

    This module handles all aspects of collision detection, including broad-phase and narrow-phase algorithms, geometric primitives, and manifold generation.

    • Box2D.Collision.IBroadPhase

      An interface identifier for broad-phase collision detection algorithms.

    • Box2D.Collision.b2AABB

      Represents an Axis-Aligned Bounding Box, used for efficient broad-phase collision detection.

      • Methods:
        • IsValid(): Checks if the AABB is valid (lower bound is less than or equal to upper bound).
        • GetCenter(): Returns the center point of the AABB.
        • GetExtents(): Returns the half-widths of the AABB.
        • Contains(b): Checks if this AABB contains another AABB b.
        • RayCast(b, e): Performs a ray cast against the AABB.
        • TestOverlap(b): Checks for overlap between this AABB and another AABB b.
        • Combine(b, e): Expands this AABB to contain two other AABBs b and e.
        • static Combine(b, e): Creates a new AABB that contains two other AABBs b and e.

    • Box2D.Collision.b2Bound

      Represents a boundary value used in broad-phase algorithms (e.g., sweep-and-prune).

      • Methods:
        • IsLower(): Checks if this bound is a lower bound.
        • IsUpper(): Checks if this bound is an upper bound.
        • Swap(b): Swaps the values of this bound with another bound b.

    • Box2D.Collision.b2BoundValues

      Stores lower and upper bound values, typically for 2D axes.

    • Box2D.Collision.b2Collision

      Provides static methods for various collision detection routines.

      • Methods:
        • ClipSegmentToLine(b, e, f, m): Clips a segment to a line.
        • EdgeSeparation(b, e, f, m, r): Calculates the separation distance between two edges.
        • FindMaxSeparation(b, e, f, m, r): Finds the maximum separation between two polygons.
        • FindIncidentEdge(b, e, f, m, r, s): Finds the incident edge for a given reference edge.
        • MakeClipPointVector(): Creates a vector of clip points.
        • CollidePolygons(b, e, f, m, r): Performs collision detection between two polygons.
        • CollideCircles(b, e, f, m, r): Performs collision detection between two circles.
        • CollidePolygonAndCircle(b, e, f, m, r): Performs collision detection between a polygon and a circle.
        • TestOverlap(b, e): Tests for overlap between two AABBs.

    • Box2D.Collision.b2ContactID

      A unique identifier for a contact point, used to persist contact information across time steps.

      • Properties:
        • key: A numerical key representing the contact ID.
        • features: An object containing detailed features of the contact (referenceEdge, incidentEdge, incidentVertex, flip).
      • Methods:
        • Set(b): Sets the contact ID from another b2ContactID.
        • Copy(): Creates a copy of this contact ID.

    • Box2D.Collision.b2ContactPoint

      Represents a single point of contact between two fixtures.

      • Properties:
        • position: The world position of the contact point.
        • velocity: The relative velocity at the contact point.
        • normal: The contact normal.
        • id: The b2ContactID for this point.

    • Box2D.Collision.b2Distance

      Provides static methods for computing the distance between two shapes using the GJK algorithm.

      • Methods:
        • static Distance(b, e, f): Computes the shortest distance between two shapes.

    • Box2D.Collision.b2DistanceInput

      Input parameters for the b2Distance algorithm.

    • Box2D.Collision.b2DistanceOutput

      Output results from the b2Distance algorithm.

      • Properties:
        • pointA: Closest point on shape A.
        • pointB: Closest point on shape B.
        • distance: The shortest distance between the shapes.
        • iterations: Number of GJK iterations.

    • Box2D.Collision.b2DistanceProxy

      A proxy for a shape, used by the GJK algorithm to query vertices and radius.

      • Methods:
        • Set(b): Initializes the proxy from a b2Shape.
        • GetSupport(b): Returns the index of the vertex furthest in a given direction.
